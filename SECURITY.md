# Security Policy

## Supported Versions

This section outlines which versions of SATURDAY are actively maintained and receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x+  | :white_check_mark: |
| 1.x     | :x:                |
| < 1.0   | :x:                |

> Only versions marked as supported receive security patches. Using unsupported versions may expose your system to vulnerabilities.

## Reporting a Vulnerability

We take security seriously. To report a vulnerability in SATURDAY:

1. **Email:** Send a detailed report to `noahtimothy14@gmail.com`. Include:
   - Affected version(s)
   - Steps to reproduce
   - Impact assessment (if known)
   - Any proof-of-concept code (optional, but helpful)

2. **Response Time:**
   - Acknowledgement within **48 hours**
   - Initial assessment within **5 business days**
   - Follow-ups will continue until the issue is resolved

3. **Remediation:**
   - If validated, a security patch or guidance will be issued
   - Advisories for critical vulnerabilities will be publicly posted after patch release

4. **Disclosure Policy:**
   - Please **do not publicly disclose** vulnerabilities before a patch is available
   - Coordinated disclosure ensures safety for all users

Thank you for helping us keep SATURDAY secure.
